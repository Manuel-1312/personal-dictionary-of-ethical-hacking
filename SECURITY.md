# Security Policy

## Reporting a Vulnerability
Si encuentras un fallo de seguridad, por favor usa uno de los siguientes canales antes de publicarlo:
1. Abre una [Security Advisory privada](https://github.com/Manuel-1312/personal-dictionary-of-ethical-hacking/security/advisories) en GitHub.
2. Envía un correo a `security@manuel-sanchez-brito.com` (si prefieres hablar en privado) y menciona claramente qué componente afecta.

Incluye pasos para reproducir, el impacto esperado y cómo podría mitigarse.

## ¿Qué ocurre después?
- Confirmamos la recepción en menos de 3 días laborables.
- Evaluamos la severidad usando CVSS o criterios equivalentes.
- Publicamos una solución o un plan de mitigación tan pronto como podamos, comunicando los cambios a través de GitHub y/o el advisory.

## Actualizaciones de seguridad
Cuando haya parches o contenido sensible actualizado, lo documentaremos en este repositorio y lo señalaremos en README/CHEAT_SHEETS para que el resto del equipo lo aplique.
