# Fun Scripts 🎉

No todo en un laboratorio tiene que oler a informe, IOC y café recalentado.

Esta carpeta existe para eso: para meter scripts pequeños, simpáticos y con un punto juguetón que hagan el repo más humano sin convertirlo en un circo. Algunos sirven para demos, otros para practicar, otros para echarte una risa, y alguno incluso termina siendo más útil de lo que parecía al principio.

La idea de `fun-scripts/` no es competir con la parte seria del proyecto, sino darle aire.
Porque sí: documentar bien está genial, pero también mola tener:
- un banner ASCII para abrir una sesión
- un nombre absurdo para un lab
- logs falsos para probar parsers
- una terminal haciendo ruido bonito porque te apetece

## Qué encontrarás aquí
Una mezcla de scripts:
- cortos
- fáciles de ejecutar
- con algo de humor o estética terminal
- y, en varios casos, sorprendentemente útiles para demos o entrenamiento

## Scripts incluidos
- `ascii_banner_maker.py` → banners rápidos para labs, notas o intros
- `lab_quote_generator.py` → frases de laboratorio con energía de terminal trasnochada
- `fake_breach_report.py` → informes ficticios para practicar reporting sin dramas reales
- `terminal_glitch_text.py` → texto con efecto raro/glitch para postureo sano
- `random_codenames.py` → nombres de operación porque "lab-3-final-final" no inspira mucho
- `phishing_subject_ideas.py` → asuntos ficticios para awareness y simulaciones internas
- `password_wordlist_mixer.py` → mini wordlists de laboratorio mezclando palabras base y sufijos
- `red_team_excuse_generator.py` → excusas absurdas, completamente innecesarias y por eso valiosas
- `log_noise_generator.py` → logs falsos para probar parsers, reportes o ideas rápidas
- `retro_matrix_printer.py` → lluvia de caracteres para decorar una terminal con dignidad dudosa

## Cuándo tiene sentido usar esta carpeta
- cuando quieras preparar una demo con un poco más de gracia
- cuando necesites datos falsos para probar un parser o un flujo
- cuando te apetezca darle personalidad a una nota o una sesión
- cuando quieras recordar que aprender también puede ser divertido

## Regla de oro
Si un script de aquí deja de ser una chorrada simpática y se convierte en herramienta real, probablemente ya no pertenece a `fun-scripts/`.
En ese caso, mejor moverlo a `python/` o a la categoría que toque.

## Recomendaciones
- Mantén esta carpeta ligera, curiosa y con sentido del humor.
- Evita que se convierta en un cajón de scripts malos “porque total, es la carpeta divertida”.
- Si un script genera salidas reutilizables, más adelante puedes guardar ejemplos en `examples/`.

En resumen: esta carpeta está para que el repo respire un poco.
Y eso, honestamente, también es buena ingeniería.
